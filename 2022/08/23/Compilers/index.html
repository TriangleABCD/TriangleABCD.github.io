0.0 什么是编译程序

A compiler is a program that can read a program in the source language, and translate it into an equivalent program in the target language.

0.1 编译过程和编译程序的结构

1.0 文法的直观概念

1.1 符号和符号串

  • 字母表：符号元素的非空有穷集合。
  • 符号串：由字母表中的符号组成的任何有穷序列，例如是字母表上的符号串。注意符号串有顺序性，和是不同的符号串。符号串可以为空，记为。若是一个符号串，则用表示的长度，显然有。

1.6 有关文法的实际应用的一些说明

不要含有害规则和多余规则：

  • 有害规则：形如 的废话规则，没用还会引起二义性。
  • 多余规则：任意句子的推导都用不到的规则，要么是非终结符不在任何规则的右部出现的形式（不可到达的），要么有的非终结符不能推导出终结符（不可终止的）。

2. 词法分析

2.0 词法分析程序的设计

  • 词法分析可以是先扫一遍源程序，将源程序变成单词序列，输出一个中间文件。也可以是将词法分析作为语法分析的一个子程序，每当语法分析需要一个单词时，就调用词法分析子程序。
  • 词法分析的输出是单词符号，分为 类：
    1. 关键字
    2. 标识符
    3. 常数
    4. 运算符
    5. 界符
  • 词法分析程序输出的单词符号可以用一个二元组表示：。单词的种别可以用整数编码表示。
  • 为什么要分离出词法分析：
    • 使得整个编译程序的结构更加的简洁清晰。
    • 改进编译程序的效率。
    • 增强编译程序的可移植性
